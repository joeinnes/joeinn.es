I made two security improvements to how Brightblur stores encrypted photo data. First, encrypted blobs are now always stored with a fixed `application/octet-stream` content type, ignoring whatever file type the client claims (since the data is encrypted anyway and the type is meaningless). Second, I added a server-side cap on how large an individual encrypted blob part can be, preventing oversized uploads.
