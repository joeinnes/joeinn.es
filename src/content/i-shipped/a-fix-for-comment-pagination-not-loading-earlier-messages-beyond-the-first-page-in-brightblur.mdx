When Brightblur's comment system was updated to use pagination, the client was only ever loading the first page and discarding the `nextCursor` needed to fetch more. Long comment threads were silently cut off at 100 comments. I wired up the cursor so clicking "load earlier" actually fetches the next page of comments.
