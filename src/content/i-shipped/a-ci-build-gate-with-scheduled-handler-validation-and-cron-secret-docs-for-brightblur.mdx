Brightblur's CI previously only ran type-checks and unit tests, so build failures — like broken Cloudflare adapter output or a bad scheduled-job injection — would only surface at deploy time. I added a full production build step to CI so these are caught early, plus a build-time assertion that the scheduled job handler is wired up correctly, and documented the CRON_SECRET environment variable in the README.
